【项目经验】基于Vue实现的换肤功能

发表于 2019-02-25 | 本文总阅读量 次
字数统计: 871 | 阅读时长 ≈ 3

简单介绍：

实现换肤功能，就我这几天搜集的思路来看，大致可以分为三种：

  • 利用elementUI框架自带的换肤组件
    • 优点：简单，快捷，直接引入组件即可
    • 缺点：只能对默认样式的element组件进行修改，如果自己写了样式覆盖组件内的样式，则无法换肤。实现的效果较为局限。

这里不推荐这种方式，如果有需求，可以自己查看elementUI官网的自定义主题项；

  • 不依托框架，自己做出较为传统换肤功能
    • 优点：可定义很多套皮肤，根据自己需求切换，覆盖范围较广
    • 缺点：需要写入多套CSS样式，主题色都是可以根据需求定义
  • 用JS创造一个link标签，引入一个早已写好的样式表
    • 优点：简单粗暴，思路简单
    • 缺点：这种方法需要一份css文件。但是，Vue项目因为有webpack的存在，所以没法预编译scss文件，这样就导致必须手写css，这样肯定难于维护。而且，样式重叠，覆盖已有样式的可能性极大，最好不要这么做。

实现思路：

1，预先写好几套不同配色的样式，

2，用JS样式给全局所有的DOM添加一个自定义属性

3，将不同的样式，对应不同的自定义属性，用JS方法setAttribute（）控制DOM的自定义属性，按需配色。
