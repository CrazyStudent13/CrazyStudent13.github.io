问题原因

出现这个问题，很可能是你的网络供应商（比如广电网）在出口防火墙上屏蔽了22端口（本地使用vpn等代理服务），这意味着你将无法访问其他主机的22端口。

github提供了一种解决方案，允许你使用443端口进行ssh连接。

因为443端口是访问https网站所必须的，大部分防火墙都会允许通过，但如果使用代理服务器可能产生干扰。
